O que decide o perfil

Só duas variáveis: IG_USER_ID + IG_ACCESS_TOKEN. O token do Instagram Login pertence a uma conta específica — hoje é o @cafecomhomensdedeus.

⚠️ Não confunda: o host de upload (cafecomhomensdedeus.com.br, no IG_UPLOAD_URL) é só onde os PNGs ficam hospedados pra Graph API baixar. Ele não define o perfil — pode continuar como está.

Passo a passo para publicar no @portaltrilhokids

1. Conta profissional + app. Garanta que @portaltrilhokids é conta Profissional (Business/Creator) e autorize-a no mesmo app Meta que já publica no cafe — fazendo o login do Instagram Business Login como o @portaltrilhokids (não como o cafe).
2. Gere o token longo (60 dias) do @portaltrilhokids — mesmo processo que você usou pro cafe, só que autorizando essa conta.
3. Pegue o IG_USER_ID da conta:
curl "https://graph.instagram.com/me?fields=user_id,username&access_token=SEU_TOKEN_NOVO"
3. Confirme "username":"portaltrilhokids" e anote o user_id.
4. Atualize as credenciais (VPS → .vps-env; host → config/env.php):
  - IG_USER_ID = user_id do portaltrilhokids
  - IG_ACCESS_TOKEN = token novo
5. ⚠️ Apague o .ig-token.json (o pulo do gato). Esse arquivo sobrepõe o IG_ACCESS_TOKEN — se ele tiver o token antigo do cafe, você continuaria postando no cafe mesmo trocando o env:
rm -f .ig-token.json
node cli.js refresh-token      # recria já com o token novo
6. Confirme antes de postar (comando que acabei de criar):
node cli.js whoami
# 👤 O token atual publica em: @portaltrilhokids (user_id: ...)
# ✅ IG_USER_ID confere com o token.
7. Teste real com 1 publicação e confira no perfil:
node cli.js carrossel --livro isaias
